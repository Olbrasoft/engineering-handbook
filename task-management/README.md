# Task Management

Guidelines for preparing and managing development tasks effectively.

## Contents

- [Task Assignment Framework](./task-assignment-framework.md) - How to write clear, actionable task specifications

## Quick Reference

### The Golden Rule

**Analyst focuses on WHAT and WHY, Programmer decides HOW.**

### Task Checklist

Before assigning any task, verify:

1. **Clear** - Can be understood without asking questions
2. **Complete** - Has user story, requirements, acceptance criteria
3. **Scoped** - Defines what's included AND what's not
4. **Testable** - Success can be objectively verified

### Minimum Viable Task

```markdown
## Summary
[What + Why in one sentence]

## User Story
As a [who], I want [what], so that [why].

## Acceptance Criteria
- [ ] Given X, when Y, then Z
```
