# Správa úkolů

Průvodce pro efektivní přípravu a správu vývojových úkolů.

## Obsah

- [Framework pro zadávání úkolů](./task-assignment-framework-cz.md) - Jak psát jasné a realizovatelné specifikace úkolů

## Rychlý přehled

### Zlaté pravidlo

**Analytik se soustředí na CO a PROČ, Programátor rozhoduje JAK.**

### Kontrolní seznam úkolu

Před zadáním jakéhokoliv úkolu ověř:

1. **Jasný** - Lze pochopit bez kladení dotazů
2. **Úplný** - Má user story, požadavky, akceptační kritéria
3. **Ohraničený** - Definuje co je zahrnuto A co není
4. **Testovatelný** - Úspěch lze objektivně ověřit

### Minimální životaschopný úkol

```markdown
## Shrnutí
[Co + Proč v jedné větě]

## User Story
Jako [kdo] chci [co], abych [proč].

## Akceptační kritéria
- [ ] Když X, pokud Y, pak Z
```
